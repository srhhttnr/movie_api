myFlix Documentation

myFlix is an API built with REST architecture. It employs logical URLs, HTTPS response codes, and returns JSON responses. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Description of data the endpoint returns Endpoint URL Method Type Format of data in body of request Format of response data
Return a list of all movies to the user /movies GET None An array of JSON objects holding data about all movies in the database
Return data (title, description, genre, director, release, image, and if featured) about a single movie by title to the user /movies/[title] GET None A JSON object holding data about a single movie
Return data (name and description) about a genre by name /movies/genre/[genreName] GET None A JSON object holding data about a genre
Return data (name, birth, death, and bio) about a director by name /movies/directors/[directorName] GET None A JSON object holding data about a director
Allow new users to register /users POST A JSON object holding data about the user to add; Example: { "firstName":"John", "lastName":"Doe", "Birthday":"2000-01-01T00:00:00Z", "Email":"johndoe@gmail.com", "Username":"johndoe123", "Password":"password123", "favoriteMovies":[] } A JSON object holding data about a user that has been added
Allow users to update their username /users/[Username] PUT A JSON object holding data about a user to update; Example: { "Username" : "jdoe123" } A JSON object holding data about the user that has been updated
Allow users to add a new movie title to their list of favorite movies /users/[Username]/movies/[MovieID] POST None A JSON object of the updated user's information
Allow users to remove a movie title from their list of favorite movies /users/[Username]/movies/[MovieId] DELETE None A text message indicating that a movie has been removed from the user's array of favorite movies
Allow existing users to deregister /users/[id] DELETE None A text message indicating that a user has been deleted
